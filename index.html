How I Approach an Audit

A repeatable workflow focused on invariants, exploitability, and high-signal reporting.

Threat Model

Map trust assumptions, roles, and external dependencies (oracles, DEXes, bridges).

Define Invariants

Formalize what must always hold: solvency, accounting conservation, and access boundaries.

Manual Review First

Audit core state transitions line-by-line before relying on automation.

Tooling & Testing

Use Slither + Foundry fuzzing/invariants to pressure-test edge cases and economic paths.

Exploit & Validate Fixes

Build PoCs where needed and verify mitigations with regression tests.

Clear Reporting

Communicate impact, reproduction steps, and concrete fixes with severity justification.
